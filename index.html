Pilot · Thumbtack Enablement

Three live modules built with Claude

A pilot test of what an Enablement designer can ship using Claude and GitHub Pages. All three modules below were built in under an hour total, by a first-time GitHub user, embedded directly in Sana. They're proofs of capability — content hasn't been validated yet and shouldn't ship to learners as-is.

How these were built

Each module is a single self-contained HTML file. The loop runs in minutes: conversation with Claude → working file → GitHub Pages → Sana embed. Total build time across all three modules: under an hour. Total prior GitHub experience: zero.

This isn't a Figma replacement — Figma still owns visual polish, design systems, and stakeholder review. What this stack does is let Enablement ship working interactive experiences — clickable, scoring, branching, content-aware — at the speed of an idea. The implication: the gap between "I wish we had a module for this" and "it's live in Sana" becomes minutes, not weeks.

What this is, and what it isn't. This was a pilot. The modules demonstrate what's possible, but they're not ready for learners — scenario content for the game needs marketplace-team validation, the refund tree needs KA accuracy review, and copy throughout needs brand and proof-reading passes. Treat these as a working capability demo, not finished curriculum. The "could-we-build-this" question is now answered yes; the next step is content rigor and scaling the pattern.
